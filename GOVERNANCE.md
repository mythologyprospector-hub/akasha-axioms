# Governance

Akasha is governed by stewardship, not by drift.

## Governance Goals

- preserve coherence
- allow experimentation
- prevent canonical entropy
- maintain traceability
- protect human accountability

## Core Rule

Strict core, creative laboratories.

Meaning:
- Core systems are held to alignment requirements.
- Labs are allowed wider exploration.
- Promotion from lab to canonical requires explicit review.

## Canonical Promotion Path

```text
idea
→ lab
→ prototype
→ canonical system
```

## Stewardship Questions

Before a system is promoted, stewards should ask:

- What is it for?
- What problem does it solve?
- What role does it serve?
- What does it consume and emit?
- Does it duplicate an existing organ?
- Is its reasoning or operation inspectable enough?

## Non-Canonical Outcome

A rejected system is not destroyed by default.
It may remain as:
- a lab
- an archive
- a note
- a failed experiment
