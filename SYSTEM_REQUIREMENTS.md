# System Requirements

A system may not become canonical in Akasha unless it declares its place.

## Minimum Declaration

Every canonical repo should declare:

- `repo_name`
- `role`
- `layer`
- `inputs`
- `outputs`
- `dependencies`
- `adapters`
- `canonical_status`
- `maturity`
- `alignment_statement`

## Minimum Alignment Statement

Every canonical repo should include a statement equivalent to:

> This system is part of the Akasha ecosystem and aligns with
> akasha-axioms, akasha-world, and akasha-constellation.

## Canonical Roles

Roles currently recognized:

- `engine`
- `forge`
- `node`
- `device`
- `academy`
- `lab`
- `graph`
- `canon`
- `runtime`
- `adapter`

Additional roles may be introduced through amendment.

## Canonical Status

A repo may be in one of these states:

- `experimental`
- `prototype`
- `canonical`
- `archived`

## Admission Test

A system must answer yes to all of the following:

1. Does it align with the axioms?
2. Does it have a declared role?
3. Does it declare inputs and outputs?
4. Can it be placed in the constellation?
5. Can its purpose be explained clearly?

If not, it does not yet exist canonically.
