# Tool Creation Doctrine

Akasha recognizes that some discoveries require new instruments.

## Principle

When a missing capability prevents further discovery, creation of the missing tool
is itself a valid and necessary response.

## Tool Creation Loop

```text
detect missing capability
→ define required role
→ scaffold system
→ test usefulness
→ integrate if aligned
```

## Preferred Properties of New Tools

New tools should be:

- role-explicit
- modular
- manifest-declared
- traceable
- interoperable
- inspectable

## Recursive Creation

Akasha explicitly permits the creation of tools that:
- improve discovery
- improve orchestration
- improve simulation
- improve evaluation
- improve generation of future tools

## Guardrail

Tool creation is not an excuse for uncontrolled proliferation.
A new tool should exist because it resolves a real structural or operational gap.
