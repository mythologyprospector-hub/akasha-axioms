# Discovery Doctrine

Akasha treats discovery as a structured process, not mere ideation.

## Discovery Sources

Discovery may arise from:

- contradiction
- repeated friction
- unexplained pattern
- missing bridge between known structures
- simulation outcomes
- cross-domain analogy
- compression gains from a simpler rule

## Preferred Discovery Loop

```text
observe
→ map
→ detect gap
→ hypothesize
→ test
→ evaluate
→ integrate
→ repeat
```

## Accepted Discovery Modes

- empirical
- computational
- simulated
- analogical
- structural
- theoretical

## Discovery Systems

Examples of systems that fit this doctrine include:

- gap detectors
- reality mappers
- anomaly detectors
- hypothesis engines
- simulation engines
- evaluation suites

## Doctrine Rule

A discovery system should not merely generate possibilities.
It should increase clarity about why a possibility matters.
